# Internationalization (i18n)

The UI ships in **Russian, English and Simplified Chinese** with a **live** switcher (no window
reload). The system is a dependency-free Svelte 5 runes store under `src/lib/i18n/`.

## How it works

- `index.svelte.ts` holds the current locale as a `$state` rune. `t(key, vars?)` reads that
  rune internally, so it is **automatically reactive** in any component template or `$derived`
  context — no subscriptions, no `useTranslation()` wrapper.
- Locale resolution order: explicit saved choice (`localStorage['cmh-language']`) →
  `navigator.language` (`ru*`→ru, `zh*`→zh, `en*`→en) → **`en`** fallback. The WebView inherits
  `navigator.language` from the OS.
- Switching is instant: `locale.set(l)` mutates the rune (and persists), re-running every `t()`.
  The switcher lives in `SettingsTab.svelte`; `initLocale()` runs in `routes/+layout.svelte`.

## Dictionary layout

`src/lib/i18n/locales/{ru,en,zh}/<namespace>.ts` — one file per namespace, assembled by each
locale's `index.ts`. Namespaces: `common`, `nav`, `titlebar`, `console`, `page`, `settings`,
`forks`, `backup`, `profiles`, `providers`, `myProviders`, `mcp`, `sync`, `plugins`, `updates`,
`glossary`, `schedule`, `analytics`, `agents`, `sessions`, `environments`, `health`, `components`,
`onboarding`.

`types.ts` derives the dictionary type from Russian — `TranslationDict = typeof ru` — and
`en`/`zh` are typed as `TranslationDict`. So **Russian is the structural source of truth**: a
missing/extra key in `en` or `zh` is a compile error (`svelte-check`).

## Pluralization

`plural(n, one, few, many)` selects the form via `Intl.PluralRules` for the current locale
(ru: one/few/many, en: one/other, zh: single form). Count-noun helpers live in the module:
`pUpdate`, `pConflict`, `pCommit`, `pAction`, `pBranch`, `pSnapshot`, `pProfile` (they read the
`common.<noun>_one/_few/_many` keys). Use them as `` `${n} ${pConflict(n)}` ``.

Also exported: `forkMode()` and `outcomeLabel()` (map backend strings → localized labels,
reading `forks.*` keys).

## Adding a string

1. Add the key to the namespace file in **all three** locales (`ru`, `en`, `zh`) with identical
   key names. Reuse a `common.*` key if one fits — don't duplicate generic verbs.
2. Use it: `import { t } from '$lib/i18n'` then `t('namespace.key')`, or
   `t('namespace.key', { name, n })` for `{var}` interpolation.
3. For `title=`/`aria-label=`/`placeholder=`, localize those too.

## Adding a locale

Add the code to `Locale` in `types.ts`, create `locales/<code>/*.ts` for every namespace,
register it in `index.svelte.ts` (`translations`, `SUPPORTED`, `getLocaleName`,
`resolveInitialLocale`) and in each `locales/<loc>/index.ts` barrel.

## Rules & gotchas

- **Never shadow `t`**: an `{#each items as t}` loop var or a `t` function parameter overrides
  the imported `t()` and silently breaks translations. Name loop vars `item`/`task`/etc.
- Keep ru/en/zh in **parity** — enforced by `npm run check:i18n`
  (`scripts/check-i18n-parity.ts`) and the vitest in `src/lib/i18n/index.test.ts`.
- `t()` is reactive only when read in a reactive position — don't cache it in a module-top
  `const`. Inside a function or `$derived` is fine.
- **Rust-side strings** (tray menu + tooltip, OS notifications) live in `src-tauri/src/i18n.rs`
  as `[ru, en, zh]` triples — add the key there, never a literal. `tr(key, lang)` /
  `trv(key, lang, vars)` are the accessors, `set_language` relabels the tray live, and the
  unit tests at the bottom of that file assert the triples.
- Out of scope: maintenance-script stdout in the console comes from the scripts themselves and
  is not translated.

## Verify

```bash
npm run check          # compile-time parity (TranslationDict) + types
npm run check:i18n     # runtime leaf-key parity across ru/en/zh
npm test               # i18n unit tests (resolve, t(), plural, helpers, parity)
```
