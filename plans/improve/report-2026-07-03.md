## Надёжность и безопасность

**1. Hardening-волна: закрыть 6 подтверждённых багов ревью 02-07** — impact HIGH, effort M
Все перепроверены сегодня, живы:
- `.mcp.json` → `cmd /C codex` инъекция метасимволов (`src-tauri/src/lib.rs:6528-6529`; комментарий сам признаёт «cmd metacharacters would not [survive]»). Fix: резолв codex.cmd напрямую или reject метасимволов.
- Утечка API-ключа: `probe_provider` (`lib.rs:4449`) шлёт `Authorization: Bearer` на base_url из myproviders.json без валидации URL/https.
- Мёртвый грид: `toggleBackground` (`SessionsTab.svelte:680-682`) не сбрасывает `maximized`; грид рендерит `activePanes` (:596, :1363-1364).
- Send-to-all в мёртвые сессии: `TerminalPane.svelte:365-368` ставит `exited=true` без `onIdChange(key,null)`.
- Fork busy-state не-RAII: `lib.rs:928-932` remove после `.await` — дроп future (F5/WebView2 recovery) стрэндит путь навсегда.
- Откат ключа провайдера теряет мигрированный legacy-слот: `lib.rs:4013-4017` (rollback удаляет только новый слот, миграция :4005 не восстанавливается).

**2. Гонка записи settings.json + частичная прошивка** — impact MED, effort M
Два писателя last-writer-wins: `write_json_atomic` из Castellyn (`lib.rs:7694-7697`) и `os.replace` из `plugin_sync.py:72-75` (срабатывает на каждом SessionStart живых CC-сессий). Одновременная запись молча теряет чужие изменения. Плюс `?`-abort внутри цикла по профилям (`lib.rs:7696`): ошибка на профиле №2 из 3 оставляет смешанное wired/unwired состояние без отката. Fix: ретрай при sharing violation + continue-on-error с отчётом по профилям.

**3. Restore-набор сессий одноразовый и хрупкий** — impact MED-HIGH, effort S
Persist-эффект (`SessionsTab.svelte:349-354`) затирает `LIVE_KEY` пустым списком сразу после маунта (комментарий :340-342 это признаёт). Ctrl+R / крэш WebView2 до нажатия «восстановить» — набор (включая `--resume` id разговоров) потерян безвозвратно. Fix: не перезаписывать LIVE_KEY, пока restore-бар не закрыт/принят.

**4. Нет single-instance guard** — impact MED, effort S
`tauri-plugin-single-instance` отсутствует (grep по src-tauri пуст). Вторая копия = двойные звуки/тосты (два poll-треда agent_status), гонки config.json, два tray-икона. Fix: плагин + фокус существующего окна.

**5. Гигиена жизненного цикла хуков** — impact MED, effort S-M
- `plugin_sync_profiles` (`lib.rs:7402-7404`) перечисляет только `.claude` + текущие имена из profiles.json → у переименованного/удалённого профиля прошитые хуки никогда не развиртуализируются и не видны в статусе.
- Выключение тумблеров не удаляет `~/.claude/hooks/*.py` (нет removal-пары к `ensure_*_script`).
- Отказ `py`-лаунчера в hook-среде CC невидим (fail-open) — статусы молча деградируют до activity-only, тумблер показывает «включено». Fix: health-проба py + индикатор в ⚙ Сессий.

**6. Эвристика blocked самосбрасывается до ответа пользователя** — impact MED, effort S-M
`agent_status.rs:191-197`: любой PTY-вывод спустя >1.5 с после Notification-хука (перерисовка промпта при ресайзе, спиннер, мигающий элемент) переводит blocked→working, гася бейдж «ждёт решения», пока промпт реально не отвечен. Fix: снимать blocked только по UserPromptSubmit/Stop-хуку или существенному burst'у байтов, а не любому выводу.

## Производительность

**7. Конвейер консольного лога** — impact MED-HIGH, effort S
Известная пара + подтверждено: `log = [...log, …]` пересоздаёт массив до 5000 строк на КАЖДУЮ строку в 8+ местах (`+page.svelte:257,266,455,1151,1704,1848,…`); backend шлёт 1 IPC-событие на строку без батча; `Console.svelte:62-65` — forced reflow (scrollHeight) на строку. Fix: единый `appendLog()` (push+splice), backend-батч ~30 мс, автоскролл через rAF.

**8. Эффективность poll-цикла статусов** — impact LOW-MED, effort S
`agent_status.rs:282-287`: read+parse hook-JSON каждого claude-pane каждые 500 мс без mtime/size-гейта (12 панелей = ~24 read+parse/с постоянно). `on_output` берёт глобальный `TRACKS`-мьютекс на каждый 32-KiB чанк каждого reader-треда. Fix: пропуск по mtime; `last_output` как AtomicU64 вне лока.

**10. Один факт — разные цвета и слова в сайдбаре и вкладке** — impact MED, effort S

**12. Пустое состояние Sessions — самодельное вместо EmptyState.svelte** — impact LOW, effort S ⚡quick-win (DRY)

## Удобство

**13. Переключение вкладки уничтожает её состояние** — impact HIGH, effort M
`+page.svelte:1982` `{#key active}` пересоздаёт компонент вкладки при каждом переключении: результаты health/balance-проверок провайдеров, раскрытые Details/ключи/редакторы — всё стирается (Sessions спасена отдельным маунтом :2101 — прецедент есть). Fix: держать смонтированными посещённые вкладки (display:none), fade — через CSS-класс.

**14. Ctrl+K-палитра молча проглатывает команды при активном прогоне** — impact MED, effort S ⚡quick-win
`+page.svelte:1571-1592`: verbs зовут `startRun`/`onStack`, каждый начинается с `if (running) return;` — комментарий прямо называет это «no-op is harmless». Набрал «apply plugins» → Enter → ничего. Fix: тост «занят: <id>» из палитры (или очередь).

**15. «Обновить всё» оставляет карточки устаревшими и идёт без прогресса** — impact MED-HIGH, effort M
`+page.svelte:1748`: авто-recheck после apply исключает `all`. `UpdatesTab.svelte:26-27`: all-карточка убрана из грида, спиннеры карточек завязаны на `running === c.id` → whole-stack прогон выглядит зависшим. Fix: recheck после «обновить всё» + подсветка текущего компонента из стрим-лога.

**16. Запуск при лимите панелей — молчаливый no-op** — impact MED, effort S ⚡quick-win
`SessionsTab.svelte:538` `if (atLimit && !v.attachId) return;` — а кнопка ▶ активна; restore/workspace при лимите молча отбрасывают лишние панели (без «восстановлено 8 из 12»). Fix: disable ▶ при atLimit + тост при частичном восстановлении.

**17. «Создать» один отсутствующий профиль = reinstall всех** — impact MED, effort M
`ProfilesTab.svelte:428`: кнопка «Создать» для missing-профиля зовёт `onAction('reinstall')` → danger-confirm с вводом слова и переустановка ВСЕХ профилей. Fix: точечный вызов установщика для одного профиля.

## Функциональность

**18. Персонализация Sessions не переносится (вне экспорта/бэкапа)** — impact HIGH, effort M
Workspaces (`SessionsTab.svelte:105-106`), favorites (:880-882), projects root, default args, ширины колонок, layout мониторов, scrollback-лимит — всё только в localStorage. Экспорт (`SettingsTab.svelte:169-192`) сериализует один `HubConfig`. Перенос на вторую машину / переустановка теряет всю настройку Sessions молча. Fix: включить в HubConfig или отдельный экспортируемый блок + захват в backup.

**19. Клавиатурный scrollback и прыжок к панели** — impact MED-HIGH, effort S
Grep `scrollLines|PageUp|copyMode` по TerminalPane пуст: прокрутка буфера — только колесо мыши (недоступно, когда TUI на alt-screen забирает wheel). Alt+1-3 занято колонками (`SessionsTab.svelte:1013`), к панели 5 из 12 — только циклический Ctrl+]/[. Fix: Shift+PgUp/PgDn + Ctrl+Home/End через `term.scrollLines/scrollToTop` в существующем `attachCustomKeyEventHandler` (:484); Alt+N → фокус панели N.

**20. Мелкие апгрейды панелей на уже собранных данных** — impact MED, effort S ⚡quick-win
- Имя панели (rename есть) не входит в `LivePane` (:339), `WsConfig` (:105), `Fav` (:880) → теряется при рестарте/workspace. Fix: +1 поле в 3 структуры.
- `term.onBell` не подписан (grep пуст) → BEL от shell/SSH-панелей (agent-status их не покрывает) пропадает. Fix: onBell → существующий unread-маркер (:234).
- `spawned_at`/`last_output` уже в Track (`agent_status.rs:74,87`), но elapsed/тишина не отдаются в StatusEvent — «работает 12 мин» vs «молчит 3 мин» не видно.

**21. Лимиты Anthropic пер-профильно + автопродолжение после сброса (запрос владельца 2026-07-03)** — impact HIGH, effort M-L
Цель: программа работает ночь/дни без присмотра; упёрлись в лимит → сами продолжаем после сброса окна.
- **Источник данных проверен**: `GET https://api.anthropic.com/api/oauth/usage` (Bearer из `<профиль>\.credentials.json` → `claudeAiOauth.accessToken`, заголовок `anthropic-beta: oauth-2025-04-20`) → `five_hour`/`seven_day` c `utilization` и `resets_at`. Ровно так уже работает statusline владельца (`C:\Users\User\.claude\statusline.py:55-89`) — DRY: тот же контракт, но пер-профильно (`.claude-*/`). Только чтение токена; refresh НЕ делаем (риск инвалидировать сессию CC). Профили без OAuth-кредов (API-key/gateway) — N/A.
- **Предупреждение заранее**: пороги **85%** (тост) и **99%** (тост+звук+OS-уведомление) — решение владельца 2026-07-03; одно уведомление на порог на окно; каналы — существующие statusSounds/statusNotify.
- **Детекция упирания**: PTY-скан в `on_output` по известным строкам («5-hour limit reached ∙ resets …», «You're out of extra usage · resets …») + подтверждение по endpoint (utilization ≥100%). Встроенного авто-резюма в CC нет (открытые фиче-реквесты anthropics/claude-code#36320, #35744, #47276); community-паттерн — claude-auto-resume (детект строки + отложенный «continue»).
- **Автопродолжение (решения владельца 2026-07-03: полный контур, ВКЛ по умолчанию, без отдельного тумблера)**: панель помечается `limited` (новое состояние точки), в `resets_at` + джиттер 30-90 с — живой панели шлём текст продолжения через существующий session_write; мёртвой — respawn `claude --resume <claudeSid>` (инфраструктура restore уже есть) + текст. Текст — по языку UI: ru «продолжай», en «continue», zh «继续» (i18n-ключ). Максимум 1 авто-продолжение на панель на окно сброса, каждый факт — в лог + тост; аварийный выключатель — только в config.json, без UI.
- **Режим после лимита — настройка** `limitMode: wait | switchProfile` (default `wait`): в режиме `switchProfile` панель перезапускается под OAuth-профилем с наименьшей utilization (<85%, linksIntact) c `--resume <claudeSid>` — осуществимо, т.к. `projects` в профилях — junction на общий `~/.claude/projects` (проверено на диске 2026-07-03: `.claude-cc1\projects → C:\Users\User\.claude\projects`); если кандидатов нет — fallback на ожидание.
- UI: чип лимитов в шапке Сессий (% + время сброса ближайшего окна), колонка в Профилях.
