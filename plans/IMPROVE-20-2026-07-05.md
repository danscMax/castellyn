## А. Надёжность свежего слоя (онбординг/плагины)

1. **MCP-шаг fail-open при нечитаемом каноне** [R] — `mcp_canon_servers()` глотает
   IO/parse-ошибку в пустой Vec (lib.rs:4034-4044), а `canon.is_empty() → "ok"`
   (lib.rs:10050): битый `.mcp.json` рисует зелёный шаг. Тот же корень — MCP-колонка
   матрицы. Фикс: отличать «файла нет/не парсится» → state `unknown`. DRY: `parse_json_bom`+`abs`.
2. **Junction-шаг не сверяет ЦЕЛЬ существующей репарс-точки** [R] — и скан (lib.rs:10009),
   и `create_settings_junction` (lib.rs:10112) судят по `is_dir(link)`: джанкшен на СТАРОЕ
   дерево или обычная папка = ложное «ok». DRY: `is_reparse_point` (lib.rs:8468) + `fs::read_link`.
3. **onbFix/onbRunAll fail-closed + run-lock для нативных фиксов** [R×2] — `onbFix` глотает
   ошибку (+page:821-823), `onbRunAll` продолжает серию после упавшего шага (+page:827-833),
   а ветки junction/managed_deploy идут мимо `running` (двойной клик = двойной UAC).
   Фикс: onbFix→boolean, `if (!ok) break`, гард `running` на нативные ветки.
4. **verify-шаг: exit-код Assert-Installation → статус чек-листа** [R+я] — код из onbSpawn
   отбрасывается (+page:820), шаг вечно «unknown» даже после провала (виден только в консоли).
5. **waitRun watchdog** [R] — ран, упавший без `run-done`, заклинивает ГЛОБАЛЬНЫЙ run-lock
   навсегда (+page:774-783, все кнопки → `common.busy`). Таймер-предохранитель:
   по истечении `pendingRun=null; reject` — лок самоосвобождается.
6. **Локализовать detail-строки онбординга** [я] — lib.rs:9988/10006/10036/10053 — английский
   хардкод при ru-UI. DRY: `tr()`/i18n.rs (как `log.plugin_managed_blocked`).
7. **Confirm перед Install-ClaudeProfiles из визарда на живой машине** [я] — onbFix (+page:817)
   зовёт reinstall без подтверждения; таб Профили гейтит его requireText-диалогом
   (+page:700-706). При `present>0` — askConfirm.

## Б. Power-user (Сессии / Плагины)

    шорткаты живут только в разрозненных title.
11. **Перенос панели в другой проект** [P] — rail-меню (:800-809) + реассайн-паттерн из
    deleteSpace (:874-887), PTY не трогается.

13. **Managed-плагины: фильтр «🔒» + авто-включение после разблокировки** [P+я] — третий чип
    к :313-318 (по образцу onlyUpdates); после unblock+deploy (+page:1677-1697) тумблер
