Лаунчер «Сессий» — 10 вариантов, где форма исчезает

Принцип (из практики tmux/zellij/wezterm + флота агентов): частый запуск — это НЕ форма. Основное действие — «ещё один такой же / здесь». Полная форма — раз на проект.

Общая модель: Space = проект = запомненный пресет (папка + среда + профиль + аргументы), задаётся один раз · запуск = «клон этой панели» или «＋ агент в проекте» без формы · папка наследуется, полным путём, не вводится · форма — только для нового/редактируемого проекта.

V1 Клон-первый (кнопка на панели)

Главное действие живёт в шапке КАЖДОЙ панели: «⧉ клон» = ещё один такой же (та же папка/среда/аргументы). Хоткей Ctrl+Shift+D. Отдельного лаунчера почти не надо.

claude · cc1E:\Scripts\Castellyn

V2 Проект-пресет + «＋» на вкладке

Каждая вкладка проекта хранит пресет; «＋» прямо на ней = мгновенный агент по пресету. Форма — только при создании проекта.

Castellyn · Claude cc1 Docx · codex

V3 «＋ Новая» = меню-память (повтор), не форма

Кнопка раскрывает список последних запусков — 1 клик «повторить». Форма спрятана в «Свой запуск…».

↻ Claude · cc1 E:\Scripts\Castellynтолько что
↻ codex E:\Scripts\Docx…5 мин
Проекты
＋ Castellyn E:\Scripts\Castellyn
⚙ Свой запуск…

V4 Командная палитра запуска (клавиатура)

Ctrl+T → печатаешь имя проекта/среды, Enter запускает. Ноль дропдаунов, ноль мыши.

cast▏
Claude · cc1 — E:\Scripts\Castellyn
codex — E:\Scripts\Castellyn
shell — E:\Scripts\Castellyn

V5 Дровер проектов (папка = контейнер)

Слева список проектов; под каждым — его агенты + «＋ агент здесь» (в папке проекта). Проект первичен.

Проекты
▾ Castellyn
E:\Scripts\Castellyn
claude cc1
codex
＋ агент здесь
▸ Docx

V6 Плитки-ярлыки «ещё один»

Домашний экран из плиток-пресетов; клик по «＋ ещё» = запуск дубля. Ярлык = «такой же ещё раз», не форма.

Castellyn
Claude · cc1
Castellyn
codex
Docx
Claude
＋новый проект

V7 Одна большая кнопка + шеврон (95% / 5%)

Гигантская «ещё агент в активном проекте» на 95% случаев; маленький ▾ — выбрать другой проект/среду.

V8 Контекст-меню на всём (ноль постоянного хрома)

Правый клик по панели/вкладке/пустому месту → «Клон», «Ещё здесь», «Новый проект». Лаунчер как бар вообще исчезает.

правый клик по панели →
⧉ Клон этой панели C-S-D
＋ Ещё Claude здесь
＋ Новый проект…

V9 Наследование cwd, как в tmux

«＋» ВСЕГДА = новый агент в папке текущей/активной панели, среда = дефолт проекта. На частом пути выбора нет вообще. Зажать Alt — выбрать среду.

→ claude в E:\Scripts\Castellyn (папка активной панели), профиль дефолт
Alt+＋ →

V10 Строка статуса снизу (vim/tmux-style)

Тонкая строка внизу окна: активный проект + хоткеи запуска. Запуск — клавиатурой, подсказки всегда на виду, хром минимален.

▸ Castellyn 5 агентов ⧉ клон C-S-D ＋ агент C-T новый проект C-S-P
Рекомендация: ядро — V9 (наследование cwd: «＋» = агент в текущей папке, без выбора) + V1 (⧉ клон на панели) как самые «tmux-верные». Сверху добавить V2 (пресет на вкладке проекта) для явного запуска в другом проекте и V3/V4 (память/палитра) для повторов. Форма — только в V6/V2 «＋ новый проект». Комбинируются в один цельный UX.
