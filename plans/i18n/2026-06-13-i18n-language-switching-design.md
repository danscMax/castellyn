# Design: Language switching (RU / EN / Simplified Chinese)

**Project:** claude-maintenance-hub (Tauri v2 + SvelteKit static + Svelte 5 runes)
**Date:** 2026-06-13
**Goal:** Add UI language switching with English and Simplified Chinese alongside the existing Russian UI.

## Reference

Mirror the proven i18n architecture already shipped in the sibling project
`Sweet Whisper` (`tauri-ui/src/lib/i18n/`): a dependency-free, Svelte-5-runes
translation store with live switching, key-parity checking, and a vitest test.
We extend it from 2 locales (ru/en) to 3 (ru/en/zh).

## Decisions (confirmed with user)

- **Locales:** `ru | en | zh` (`zh` = Simplified Chinese only; `zh-*` OS locales all map to `zh`).
- **Default / detection:** OS auto-detect. Resolution order: explicit saved choice
  (`localStorage`) → `navigator.language` (`ru*`→ru, `zh*`→zh, `en*`→en) → **`en`** fallback.
- **Switching:** live, no window reload — `locale.set()` mutates `$state`, which
  re-runs every `t()` consumer synchronously (Svelte 5 runtime).
- **Persistence:** `localStorage` key `cmh-language` (mirrors `theme.ts` / `cmh-theme`).
  The Rust backend does not need the language, so no `HubConfig` field.
- **Scope:** entire UI — tabs, dialogs, buttons, `title` tooltips, toast/flash
  messages, pluralization. **Out of scope:** stdout text streamed from the
  maintenance scripts into the Console (originates in the scripts themselves).
- **Translations:** EN + zh-Hans generated by the assistant; user reviews (esp. zh).

## Module layout (`src/lib/i18n/`)

| File | Responsibility |
|---|---|
| `types.ts` | `export type Locale = 'ru' \| 'en' \| 'zh'` + `TranslationDict` interface (nested by domain). |
| `index.svelte.ts` | `$state` locale; `t(key, vars?)`; `resolveInitialLocale(saved, navLang)`; `getInitialLocale()`; `locale` helper (`current`/`set`/`supported`); `getLocaleName()`; `plural(...)` (locale-aware); `LANG_STORAGE_KEY='cmh-language'`; `initLocale()`. |
| `index.ts` | Barrel re-export (so bare `$lib/i18n` resolves the `.svelte.ts`). |
| `locales/ru.ts` | Russian — the current hardcoded strings. |
| `locales/en.ts` | English. |
| `locales/zh.ts` | Simplified Chinese. |

`TranslationDict` top-level domains (one per tab/area):
`common`, `nav` (sidebar), `titlebar`, `forks`, `sync`, `plugins`, `mcp`,
`providers`, `profiles`, `backup`, `schedule`, `updates`, `settings`, `console`,
`dialogs`, `toasts`.

## Pluralization

The existing `i18n.ts` exports `plural(n, one, few, many)` + `pUpdate/pConflict/
pCommit/pAction/pBranch/pSnapshot/pProfile` (Russian-only). Replace with a
locale-aware scheme:

- Plural word forms live in the dictionaries as `<noun>_one/_few/_many` keys.
- A `plural(n, forms)` helper picks the category via `Intl.PluralRules(locale)`:
  - ru → `one | few | many | other`
  - en → `one | other`
  - zh → `other` (single form)
- Helper maps category → the right dict key; `zh` simply always uses one form.

The label maps `forkMode()` and `outcomeLabel()` from the current `i18n.ts` move
into the dictionaries (`forks.mode_*`, `forks.outcome_*`); `outcomeLabel` keeps
returning `{label, cls}` but reads the label from `t()`.

## Language switcher UI

In `SettingsTab.svelte`, a new card next to **Тема**: label + 3-way control
(`Русский` / `English` / `简体中文`). On change: persist to `localStorage` and
call `locale.set()`. No reload.

## Initialization

`+layout.svelte` already calls `initTheme()` in `onMount`. Add `initLocale()`
beside it. The i18n module reads `localStorage`/`navigator` lazily at module load
(guarded for SSR-undefined), so `t()` is correct from first paint; `initLocale()`
is a no-op hook kept for symmetry / future side effects.

## Migration

Replace every hardcoded Cyrillic string across ~24 components with `t('domain.key')`,
adding the matching entry to **all three** locale files simultaneously. Order
by tab to keep dictionaries coherent. `+page.svelte` (nav/tab labels) and
`Sidebar.svelte` first, then each tab + its dialogs.

## Quality gates

- `scripts/check-i18n-parity.ts` — verifies ru/en/zh share an identical set of
  leaf key paths (adapted from Sweet Whisper). Wired as npm `check:i18n` (via `tsx`).
- `src/lib/i18n/index.test.ts` (vitest) — parity across the three dicts +
  `resolveInitialLocale` cases (ru/en/zh/de/empty) + `t()` interpolation + missing-key fallback.
- `npm run check` (svelte-check) and a production `npm run build` must stay green.

## Non-goals

- No i18n of maintenance-script stdout.
- No backend (Rust) changes.
- No translation-management tooling beyond the parity script.

  `glossary.ts` migrated; **zero Cyrillic left in any component** (only dicts,

- Live switcher in `SettingsTab` (Русский / English / 简体中文); init in
  `+layout.svelte`; persisted to `localStorage` `cmh-language`; OS auto-detect → en fallback.
- **Extra:** explanatory `title=` tooltips added (all 3 languages) to controls
  that lacked them, plus per-section nav tooltips in the sidebar.
- Verified green: `npm run check` (svelte-check 0/0), `npm test` (26 passed),
  `npm run build`, and `npm run check:i18n` → **781 keys identical across ru/en/zh**.
- Gotcha recorded: never shadow the imported `t` with an `{#each … as t}` loop
  var or a `t` param (hit and fixed in ScheduleTab, ToastHost, setTheme).
