# Живой смоук Run 1 (2026-07-03) — находки и фиксы

Смоук Run 1 (ветка `auto-loop/improve-0703`) владельцем в живом приложении. Гейты были зелёные,
но живой запуск вскрыл 6 багов + by-design-вопросы + направление крупной переделки. **Merge держим,
пока не починим 6 багов и не пройдёт ре-смоук.** Корни верифицированы 3 суб-агентами по коду.

## 🔴 Баги — чинятся ДО merge (root cause, без костылей)

### BUG 1 — Shift+Enter не вставляет новую строку в терминалах
- **Корень:** `src/lib/components/TerminalPane.svelte:492-523` (`attachCustomKeyEventHandler`) не имеет ветки Enter;
  xterm 6.0.0 ветвит Enter только по `altKey`, поэтому и Enter, и Shift+Enter отдают `\r` (CR) → Claude сабмитит.
  Claude вставляет перевод строки только на `\x1b\r` (ESC+CR) — ровно то, что шлёт Alt+Enter.
- **Фикс:** ранняя ветка в хендлере: на `Enter && shiftKey` слать `\x1b\r` (через `onInput`/`sessionWrite`), `return false`.
- **Трейд-офф:** `\x1b\r` уходит на всех харнессах — для Claude верно, в shell безвредно (= Alt+Enter).

### BUG 2 — Вставка дублируется ×2
- **Корень:** два пути вставки на Ctrl+V. Наш перехват `TerminalPane.svelte:502-505` → `paste()` (`:235-246`) →
  `term.paste`. Наш `return false` НЕ зовёт `preventDefault`, поэтому браузерный paste проходит и срабатывает
  встроенный `handlePasteEvent` xterm → второй экземпляр.
- **Фикс:** `e.preventDefault()` в ветках Ctrl+V и Ctrl+Shift+V перед `return false`. Наш `paste()` оставить
  (нужен для WebView2 / синтетического Ctrl+V / правого клика / сниппетов — они уже single-fire).
- **Трейд-офф:** полагается на подавление paste через `preventDefault(keydown)` (документированное поведение
  WebView2). Фолбэк, если не сработает: capture-phase `paste`-листенер с `preventDefault + stopImmediatePropagation`.

### BUG 3 — Символы статус-лайна = тофу/квадраты
- **Корень:** `TerminalPane.svelte:416` `fontFamily: "'Cascadia Code','Consolas',monospace"` — ни один не покрывает
  Nerd Font PUA (иконки U+E000–F8FF, U+F0000+, powerline). В проекте 0 `@font-face`, нет вшитых шрифтов.
- **Фикс:** вшить `CaskaydiaCove Nerd Font Mono` (regular+bold, .woff2/.ttf) в `static/fonts/`, `@font-face`,
  префиксом в `fontFamily`; грузить ДО `term.open` (глифы меряются на open; иначе WebGL-атлас закэширует тофу).
- **Трейд-офф:** +~1–3 МБ бандла. Субсеттинг не помогает (нужен весь диапазон иконок). Альтернатива «поставь
  шрифт сам» = костыль (ломается из коробки). Шрифт SIL OFL — redistributable.

### BUG 4 — Ложное «Агент закончил» на ещё работающем агенте
- **Корень:** `agent_status.rs:26` `ACTIVITY_IDLE_MS=4000`; ветка `Some("working")` (`:291-297`) на паузе вывода
  >4с возвращает `idle`, хотя hook ещё `working` и Stop не было. Транзишн `working→idle` (`:411-424`) →
  `notify_transition` (`:229-259`) шлёт «done». Подавлено при фокусе (`:235-241`) → бьёт, только когда окно
  не в фокусе (оттого и заметно). Настоящие ходы имеют паузы >4с (мышление, длинный tool-call, сеть).
- **Фикс:** гейтить `completed` на hook-idle (добавить в `StatusEvent` поле `hook_idle`, требовать реального Stop);
  heal-бэкстоп для `working` удлинить до ~30–45с (короткое 4с-окно оставить только для hookless-ветки codex/opencode).
  Фронт независимо выводит «done» из того же транзишна — `SessionsTab.svelte:259-262`, править тоже.
- **Трейд-офф:** у codex/opencode нет hook → для них «done» остаётся по активности (гейт применять только к
  панелям с hook_state). Упавший Stop на реальном ходе Claude → нет тоста (греется по бэкстопу). Правильный
  перекос: лучше пропустить «done», чем ложно сообщить.

### BUG 5 — Codex запускается с Claude-флагом `--dangerously-skip-permissions`
- **Корень:** `SessionsTab.svelte` — одно общее поле `lArgs`/`defaultArgs` (`:103,:831`) на все харнессы; сеид-эффект
  (`:835-837`) не знает про `lEnv`; переключение вкладки (`:1208`) не пере-сеидит. `session_spawn` (`lib.rs:9440-9449`)
  дословно дописывает флаг к codex → `error: unexpected argument`.
- **Фикс:** сеид per-harness — `lArgs = lEnv==='claude' ? defaultArgs : ''`; хендлер `selectEnv` сбрасывает
  `argsTouched=false` при смене вкладки. Codex/opencode/shell → пусто (правильный опасный флаг codex =
  `--dangerously-bypass-approvals-and-sandbox`/`--yolo`, но авто-инжект «отключить sandbox» — footgun; впишет сам).
- **Трейд-офф:** нет персистентного per-harness дефолта (минимальный фикс). Полный вариант (`Record<Env,string>` +
  миграция) — отложить.

### BUG 6 — Чип лимитов (ProfileUsageBadge) мигает
- **Уточнение:** это НЕ чип 21d (тот ещё не сделан — `limits-status` никто на фронте не слушает). Мигает
  `ProfileUsageBadge` compact в баре панели (`TerminalPane.svelte:623`), питается `read_profile_usage`, не `limits.rs`.
- **Корень:** `read_profile_usage` (`lib.rs:5073-5101`) на любом сбое ре-фетча возвращает `None`; кэш обновляется
  только при успехе, устаревшая запись не отдаётся. Бейдж рендерит по non-null (`ProfileUsageBadge.svelte:52`) →
  на `null` исчезает. 60с-интервал бейджа и 60с-TTL кэша рассинхронены → осцилляция под нагрузкой.
- **Фикс (DRY, бэкенд):** на неудачный ре-фетч отдавать последнее хорошее из кэша (кап ~15м); 401 → трактовать
  как «очистить» (по образцу `limits.rs:219-233`), прочие ошибки — «держать». Фронт: `if (next) u = next` (не обнулять).
- **Трейд-офф:** разлогиненный профиль показывает stale до капа — приемлемо и строго лучше мигания.
