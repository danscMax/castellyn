# Волна 4 herdr-порта: редизайн верстки «Сессий» (layout)

Статус: **утверждён** по итогам pre-implementation interview 2026-07-04 (владелец выбрал 4 опции).
Реализация — инкрементально по волнам, каждая: ветка → гейты → живой смоук → merge.
herdr = AGPL-3.0: копируем UX-модель, НЕ код/ассеты.

## Проблема
Постоянный хром над сеткой терминалов занимает ~5 строк (header + launchhead + phrase + favs +
workspaces). При 4+ панелях терминалам мало высоты; нет быстрого обзора/перехода между агентами;
при многих проектах панели свалены в одну общую сетку.

## Решения (из интервью 2026-07-04)
1. **Лаунчер (Q1=A):** постоянные строки → кнопка **«＋ Новая сессия»** + поповер с фразой-
   конструктором; избранное — компактные чипы рядом.
2. **Левый рельс агентов (Q2=A):** список запущенных панелей (среда + папка + статус-точка +
   чип лимита), клик = фокус панели; сворачиваемый.
3. **Полные herdr-spaces (Q3=B):** «Наборы»/workspaces → таб-полоса проектов; панель принадлежит
   space; переключение space показывает свой набор панелей (НЕ убивая PTY); «＋» = новый space.
4. **Инкрементально по волнам (Q4=A).**
5. **21d чип лимитов:** в шапке каждой панели (`ProfileUsageBadge` рядом с профилем).

## Волны

### Волна 1 — компактный лаунчер + чип лимитов (низкий риск)
- Свернуть header/launcher в одну строку: `[＋ Новая сессия ▾]` + чипы избранного + (при >1 панели)
  поиск / send-all / broadcast / колонки 1·2·3 / ⋯ / закрыть-все.
- «＋ Новая сессия» открывает **поповер-панель** с текущей фразой-конструктором (envseg + phrase +
  секция ⚙-настроек внутри). **Reuse:** вся логика фразы (`lEnv/lProfile/lLoc/lFolder/lArgs/
  launchPhrase`) переезжает в поповер БЕЗ изменений; нужен лишь контейнер-поповер (лёгкая абсолютно-
  позиционированная панель, закрытие по клику-вне / Esc).
- Настройки (projects root, default args, status hook, limitMode, SSH-серверы) — внутри поповера.
- Строка «Наборы»/workspaces временно остаётся (уедет в Волну 3 как spaces-табы).
- 21d чип: `ProfileUsageBadge` (уже есть) в шапке панели рядом с профилем.
- Файлы: `SessionsTab.svelte` (шапка/лаунчер), новый `SessionLauncherPopover.svelte` (если чище),
  `TerminalPane.svelte` (чип). i18n ru/en/zh.
- Смоук: запуск сессии из поповера; обе темы; терминалам заметно больше высоты.

### Волна 2 — левый рельс агентов
- Узкая колонка слева от сетки: по панели — иконка среды, папка/имя, статус-точка
  (working/blocked/done/idle), чип лимита; клик = фокус (scrollIntoView + focus term); сворачивается.
- **Reuse:** `panes`, `agentStates`, статус-точки/цвета (уже есть), фокус-логика (Ctrl+]/[).
- Файлы: `SessionsTab.svelte`, новый `AgentRail.svelte`. i18n.
- Смоук: 4+ панели, клик по агенту фокусирует, статусы совпадают с точками панелей.

### Волна 3 — полные herdr-spaces (смена модели)
- **Модель (безопасная для PTY):** добавить поле `space` каждой панели; **плоский список панелей
  остаётся**, сетка **фильтрует по активному space** — все PTY смонтированы, переключение space
  меняет только фильтр (панели не убиваются). Отвергнута альтернатива «раздельные массивы панелей
  на space» — крупнее и рискованнее.
- Таб-полоса сверху: `[● proj-A][● proj-B][＋]`; активный = свой набор; roll-up статуса на табе.
  «Наборы»/workspaces мигрируют: сохранённый space = именованный набор фраз для пере-открытия.
- Новые панели создаются в активном space. Перемещение панели между spaces — опционально.
- Файлы: `SessionsTab.svelte`, `sessionPrefs.ts` (spaces в персист), i18n.
- Смоук: создать 2 space, панели в каждом, переключение сохраняет живые PTY.

## DRY / переиспользовать
Фразу-конструктор целиком (В1), статус-точки/`agentStates` (В2), workspaces-персист + `sessionPrefs`
sidecar (В3), `ProfileUsageBadge` (чип). Не изобретать поповер/фокус/статусы — они уже есть.
