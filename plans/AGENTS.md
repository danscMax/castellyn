reference — read for intent and history, not as current contracts.

## Local Contracts

- Historical: a plan here describes intent at its time and may not match current code — verify
  against the code before acting on it
- Most plans are tracked in git; a few subfolders are gitignored (`plans/audit/`,
  `plans/ui-review/`, `plans/redesign/screens/`) — those hold disposable audit/screenshot output
- Do not treat a plan as a live contract that overrides the actual implementation or the DOX chain

## Work Guidance

- When a plan ships, the durable outcome belongs in code + the relevant AGENTS.md / `docs/`, not
  in a growing plan file
